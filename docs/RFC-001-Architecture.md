# RFC-001: Архитектура TGFlow

- Тип: RFC
- Статус: Active
- Владельцы: TGFlow Dev
- Дата: 2025-12-28

## Цели

- Обеспечить надежную массовую рассылку сообщений в Telegram с нескольких аккаунтов.
- Локальное хранение всех данных пользователя.
- Удобный UI на PyQt6.

## Компоненты

- Desktop (PyQt6):
  - UI/настройки.
  - `BroadcastWorker`: отправка сообщений с использованием Pyrogram.
  - Логи: HTML-журналы локально.

## Логирование

- Локально: HTML-логи по сессиям хранятся в директории `broadcast_logs/`.

## Безопасность

- Все сессии Telegram хранятся локально в директории пользователя.
- Прямое взаимодействие с Telegram API без промежуточных серверов.
